# DX Checklist

> From: Vibe Coding Bible, Vol 1 â€” Appendix R

For any tool, library, or API you build:

## Onboarding

- [ ] First success in < 5 minutes
- [ ] Zero config to start
- [ ] Clear "next steps" after hello world

## Documentation

- [ ] Every public API documented
- [ ] Working examples for common use cases
- [ ] AI-parseable structure (consistent format)
- [ ] Error messages link to relevant docs

## Errors

- [ ] Errors explain WHAT, WHERE, WHY, HOW
- [ ] Suggestions are actionable
- [ ] Links to docs/solutions included

## Consistency

- [ ] Naming conventions documented and followed
- [ ] < 3 ways to accomplish any task
- [ ] Patterns predictable across the API

## AI-Readiness

- [ ] Strong typing (TypeScript, Python type hints)
- [ ] Self-contained examples (copy-paste friendly)
- [ ] Consistent patterns AI can learn
